pytest.ini

    Purpose: Configuration for the pytest testing framework.

    Common settings:

        testpaths = tests – where to look for tests.

        addopts = -v --cov=src – verbose output + coverage report.

        python_files = test_*.py – pattern for test files.

    Why needed: Standardises how everyone runs tests and avoids typing long command‑line flags.



mypy.ini

    Purpose: Configuration for the static type checker mypy.

    Why needed: ETL pipelines often deal with messy data types. Static typing catches mismatches early (e.g., passing a string to a function expecting a date).

    Typical content:
    text

    [mypy]
    python_version = 3.11
    ignore_missing_imports = true
